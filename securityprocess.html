Security Terms and conditions

COMMITMENTS BY THE SERVICE PROVIDER

The obligations regarding the security and confidentiality of personal data, where applicable, shall be passed on to potential suppliers or subcontractors of Parties having access to such data. It is the responsibility of the Service Provider to implement appropriate technical and organizational measures to protect personal data against accidental or unlawful destruction, accidental loss, unauthorized alteration, dissemination or access, in particular In the context of the transmission of this data through a non securized network as well as against any form of unlawful treatment.

OpenID Connect Protocol Security Requirements

The Service Provider shall implement the technical and organizational security measures necessary to ensure, within its perimeter:

  • Non-disclosure of functional and technical data exchanged under the protocol to an unauthorized third party,
  • Putting in place measures to prevent their escape in case of intrusion,
  • The confidentiality and integrity of the secrets exchanged (passwords, cryptographic keys…)

The Service Provider also meets the following requirements:

  • Implement the necessary security measures to ensure the secure storage of the secret client and Open Id Connect client,
  • Systematically validate all input data, if possible by using blank lists, to prevent, for example, their manipulation by inserting specific characters. In particular: check the format of authorization and access tokens (token_Id),
  • Check the nonce parameter of the access token. The latter is made compulsory during the call to XConnect to avoid the replay of the requests,
  • Check the expiration date of the access token,
  • Check the domain name of the server returned with the name used for the server-to-server call (FS -> FD call).

Monitoring and awareness

The Service Provider implements an advanced watch on its perimeter to detect cyber criminal attacks on services related to XConnect. In the event of an attack, it undertakes to alert FDS and all the partners of the security chain. The Service Provider trains and sensitizes the players under its authority to the security and stakes of XConnect (including developers and target agents using XConnect).

Global recommendations for Secure Implementation of Digital Services

The Service Provider is recommended to use standard recommendations for securing web applications in particular:

  • Apply defense principles to application software and hardware architectures. The implementation of these principles by appropriate measures is to be considered at the design stage, in view of the risks and threats to which the application will be exposed,
  • Secure the administration process using secure protocols and restrict administrative tasks to only duly authenticated and authorized administrative stations,
  • Apply the principle of least privilege to all elements of the system ("anything that is not explicitly authorized is prohibited by default"),
  • Systematically check the input data of the queries, whether functional or technical and from whatever source.

COMMITMENT BY FDS

Regulatory Compliance and security Measures

FDS implements the appropriate technical and organizational security measures to protect the data processed and stored within the framework of the service, in the light of the security objectives identified following the analysis of security risks. In particular, these measures concern

  • Systematic monitoring of all input parameters to reduce the risk of injection. FDS implements mechanisms to block customers in case of repeated failures in order to avoid brute force attacks. This can go as far as disconnecting a service provide in the event of a critical threat,
  • The robustness of the secrets, their safe storage and transmission as well as their regular renewal
  • In general: the application of defense principles in depth, particularly in terms of managing access rights to various components of the system (reverse proxies, application and data servers, etc.).
