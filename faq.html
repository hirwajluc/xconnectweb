How do I reconciliate the pivotal Identity returned by X Connect with my existing users base?

1st case : The user creates a new account at Service Provider by using X Connect.

You take all or part of the data (the sub and pivotal identity) and you do create an account on your system by telling the user that they will have to use X Connect to access that account onwards (this is our preconisation at XConnect).

2nd cas : the user would like to use X Connect while already having an account.

The Service Provider has to do what is called a reconciliation: this happens by asking the user a secret that you share with them (Customer ID, last nvoice value,â€¦). or by asking them to enter their login and password

The sub is then associated to an exiting account. Next time the user logs in with X Connect, the Service Provider fetches the sub and display the corresponfing user account at their side.

Do it get the same sub if the user connects through different Authentication Providers?

For the time being no but this will soon be the standard. This does not pause any problem today because the SP would generally integrate a specific Authentication Provider.

For any given user do all SP receive the same sub?

No. X Connect returns a different sub for the same user connecting to a different SP .

Is the received sub linked to the user identity?

No. the sub is randomly generated.

What is the length od the syb field?

The length as defined in the standardd is 255 caracters maximum.

Can I initiate the OpenID COnnect Kinematic on the authorize endpoint and request for userInfo later?
